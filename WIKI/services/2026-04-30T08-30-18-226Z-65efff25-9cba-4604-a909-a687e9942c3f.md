## Summary
Deck is the operational dashboard of AiAgentNerd, providing a live internal web interface for real-time monitoring and control of the multi-agent system. Accessible at `deck.aiagentnerd.com`, it centralises visibility into agent activity, task status, system health, and event logs. It serves as the primary bridge between human operators and the autonomous agent system, offering administrative controls and live updates via polling or WebSocket.

## Key Concepts
- **User Panel**: Displays active users and their running task counts.
- **Agent Office View**: Shows all agents, their current task, status, and the LLM model in use.
- **Task Panel**: Lists active and queued tasks with controls to stop or retry.
- **Activity Feed (Ship Log)**: Chronological log of system events, agent actions, and errors.
- **System Monitoring**: Real‑time CPU, memory, health metrics, and alerts.
- **Structured Data Model**: Consumes `users[]`, `agents[]`, `tasks[]`, `activity[]`, and `system_status`.
- **Interaction Model**: Click an agent to view its task, filter by user, or drill into task details.

## Practical Use
- Monitor the entire AiAgentNerd system in real time.
- View which agents are running and what they are doing.
- Manage task queues: stop or retry tasks directly from the interface.
- Filter activity by user to focus on specific operations.
- Keep an eye on system health and receive alerts directly.

## Implementation Notes
- Runs as a web‑based **internal dashboard** at `deck.aiagentnerd.com`.
- Data is supplied through structured JSON-like objects (`users`, `agents`, `tasks`, `activity`, `system_status`).
- Real-time updates are achieved via polling or WebSocket connections.
- Agent state changes, task updates, and activity events are streamed live.
- The interface follows a single-page dashboard layout, divided into distinct panels for each data domain.
