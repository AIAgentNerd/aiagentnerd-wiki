## Summary
AionUi assistant response describing Deck, the operational dashboard of the AiAgentNerd multi-agent system. This interaction confirms that the Deck interface is recognized and can be surfaced to users via the AionUi chat layer.

## Key Concepts
- Deck is a single‑page web application running at `deck.aiagentnerd.com`
- Five‑panel layout: User Panel, Agent Office View, Task Panel, Activity Feed (Ship Log), System Monitoring Panel
- Real‑time updates through polling or WebSocket
- Primary control and visualization layer for Hermes task execution
- Role: operator’s entry point into the system

## Practical Use
- Monitor the entire agent crew in real time
- View each agent’s current task, status, and model
- Filter tasks by user; see active and queued job IDs and types
- Administrators can stop or retry tasks directly from the interface
- Observe activity log and system health metrics

## Implementation Notes
- Endpoint: `deck.aiagentnerd.com`
- Consumes structured data objects: `users[]`, `agents[]`, `tasks[]`, `activity[]`, `system_status`
- Interaction: click an agent to see its current task; click a user to filter tasks; click a task for details
- Refresh mechanism: polling or WebSocket for live state streaming
