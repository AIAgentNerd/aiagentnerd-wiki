# Aionui 2026 04 30T08 30 54 915Z D7c30494 822a 41c3 Adb3 06da55b35bf0

## Summary
Deck is AiAgentNerd’s internal operational dashboard, accessible at `deck.aiagentnerd.com`. It provides real-time monitoring and management of the multi-agent system through a single-page interface. Deck serves as the primary control interface, the visualization layer for Hermes task execution, and the operator’s entry point into the system.

## Key Concepts
- **Single-page web dashboard** for real-time system oversight.
- **Five integrated panels** covering users, agents, tasks, activity logs, and system health.
- **Live updates** via polling or WebSocket connections.
- **Agent visualization** – displays each agent’s current task, status, and assigned model.
- **Operator controls** – supports administrative actions like stop/retry on tasks.

## Practical Use
- Operational interface for monitoring the multi-agent environment.
- Allows filtering tasks by user, inspecting agent states, and tracking active/queued workflows.
- Provides real-time visibility into Hermes task execution and system health alerts.
- Used by operators to intervene in task lifecycles (stop, retry).

## Implementation Notes
- **URL**: `deck.aiagentnerd.com`
- **Layout panels**:
  - **User Panel** – lists users and active task counts; supports filtering.
  - **Agent Office View** – shows all agents, each with current task, status, and model.
  - **Task Panel** – active/queued tasks with ID, type, user, status; admin stop/retry.
  - **Activity Feed (Ship Log)** – chronological log of events, agent actions, errors.
  - **System Monitoring Panel** – live CPU, memory, health, alerts.
- **Real-time updates**: polling or WebSocket.
- **Role**: acts as the visualization and control layer for Hermes, not a user-facing product.
