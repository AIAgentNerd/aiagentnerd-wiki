# Aionui 2026 04 30T09 34 15 091Z 4d760d59 A6b6 4af0 B0ef 5d2e6ffe86cf

## Summary
This note documents Deck, the operational dashboard of the AiAgentNerd system, as captured in an AionUI chat exchange. Deck is a single-page web application that provides real-time monitoring and administrative control over the multi-agent crew through five integrated panels.

## Key Concepts
- **Deck**: Primary operational dashboard and control interface for AiAgentNerd
- **Agent Office View**: Visual crew representation showing each agent's current task, status, and active model
- **Ship Log**: Chronological activity feed of system events, agent actions, and errors
- **Real-Time Updates**: Live state synchronization via polling or WebSocket

## Practical Use
- Serves as the main operator entry point at `deck.aiagentnerd.com` for Hermes task execution
- Enables monitoring of users, agents, tasks, and system health from a unified interface
- Supports direct administrative actions including stopping and retrying tasks

## Implementation Notes
- **URL**: `deck.aiagentnerd.com`
- **Type**: Single-page web application
- **Data Consumed**: Structured data for users, agents, tasks, activity, and system status
- **Panel Layout**:
  1. **User Panel** — user list with active task counts and user-based filtering
  2. **Agent Office View** — primary interaction area displaying agent state and assigned models
  3. **Task Panel** — active and queued tasks (ID, type, user, status) with admin stop/retry controls
  4. **Activity Feed (Ship Log)** — chronological log of system events and agent actions
  5. **System Monitoring** — CPU, memory, health metrics, and runtime alerts
- **Interaction Patterns**:
  - Click an agent to reveal its current task
  - Click a user to filter the task list
  - Click a task to open detailed view
  - Admin controls for stopping or retrying jobs
- **Update Mechanism**: Polling or WebSocket for live task states and agent activity streams
