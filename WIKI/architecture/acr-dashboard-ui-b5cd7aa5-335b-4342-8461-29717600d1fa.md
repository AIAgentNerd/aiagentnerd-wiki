# Openwebui Acr Dashboard Ui B5cd7aa5 335b 4342 8461 29717600d1fa

## Summary
Layout blueprint for the ACR (Agent Control Room) Dashboard UI. Defines a full-viewport dashboard shell with a header, sidebar, split main area (Control Panel and Results), and footer status bar. Includes a React + MUI implementation skeleton and responsive grid configuration.

## Key Concepts
- **Header**: Global branding and navigation; logo left, title right, user menu (profile, logout)
- **Sidebar**: Contextual navigation for Dashboard, Jobs, Settings, and Help
- **Main Area**: Vertically split into Control Pane (job controls, filters, status) and Results Pane (content table/grid with timestamps, confidence scores, media preview)
- **Footer**: Sync status indicator, toast/snackbar area, about link
- **Responsive Behavior**: Panels stack on narrow viewports via CSS grid/flexbox
- **Theme Support**: Dark/light mode toggle

## Practical Use
- Base layout for the agent control room job-management dashboard
- **Control Panel** actions: Start Job, Stop, Refresh; Filters (All, Top); dynamic Status and Last Updated display
- **Results Panel** columns: Time, ID, Confidence score, Media Preview
- **Status Feedback**: Toast notifications for success/failure; progress spinners next to running jobs
- **Export**: CSV/PDF export button in Results pane for audit trails

## Implementation Notes
- **Framework**: React with Material-UI (MUI)
- **Entry Point**: `src/pages/ACRDashboard.tsx`
- **Layout Specs**:
  - Root container: `flexDirection: column`, `height: 100vh`
  - Header: `height: 60`, `elevation: 3`, `padding: 0 20px`
  - Sidebar: `width: 200`, `elevation: 1`, vertical padding `20px`
  - Main body: `flexGrow: 1`, `overflow: hidden`
  - Main grid: MUI `Grid container` with `spacing: 2`, `padding: 20`
  - Control Panel: `xs={12} md={4}`, `height: 100%`, `padding: 16`
  - Results: `xs={12} md={8}`, `height: 100%`, `padding: 16`
  - Footer: `height: 30`, `elevation: 0`, `padding: 0 20px`
- **Accessibility**: Proper `aria` labels for menus, buttons, and table cells
- **State Feedback**: Spinners/progress bars next to running jobs; toasters for success/failure messages
