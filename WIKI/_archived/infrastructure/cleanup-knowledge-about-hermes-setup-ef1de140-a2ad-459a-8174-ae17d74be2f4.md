## Key Concepts
- **Knowledge Cleanup**: An automated scan that identifies redundant or overlapping documentation across the wiki and RAW ingestion folders.
- **Duplicate Detection Heuristics**: Matches based on duplicate filename, near-duplicate title, similar topics, overlapping paths, and OpenWebUI autogenerated filename penalties.
- **Canonical Selection**: Preferred version chosen from duplicates using criteria: RAW source over compiled WIKI, non-archived status, cleaner title, and best category fit.
- **Confidence Levels**: Rated high, medium, or low based on similarity strength and path overlap.
- **Report-Only Mode**: Default behavior that surfaces recommendations without file modifications; explicit merge commands are required to alter files.

## Practical Use
- Run cleanup to deduplicate wiki knowledge before major reorganizations or compilations.
- Prefer RAW sources over compiled WIKI notes when selecting canonical versions during merges.
- Treat OpenWebUI autogenerated filenames (UUID suffixes) as lower-quality canonical candidates.
- Review canonical confidence levels before executing merges; low-confidence suggestions require user choice.
- Archive originals after merging into the selected canonical file.
