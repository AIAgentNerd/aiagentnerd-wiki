# Concepts Deck

## Summary
Deck is the primary control interface for the AiAgentNerd multi-agent system. It provides a unified operational view for visualizing agents, monitoring tasks, managing users, and observing real-time system activity through a ship metaphor.

## Key Concepts
- **Ship metaphor**: AiAgentNerd is the ship, Deck is the helm, agents are the crew, tasks are missions, and the activity feed is the ship’s log.
- **Agent visualization**: Real-time status of all agents and their current work.
- **Task monitoring**: Tracking active and queued tasks across the system.
- **User management**: Control of users and permissions within the interface.
- **Activity feed**: Chronological event stream of system actions for oversight and debugging.

## Practical Use
Deck serves as the operator’s console for interacting with the multi-agent system. It allows an administrator or operator to see what every agent is doing, monitor task progress, manage users, and follow the live ship log. It is the central glass through which the system’s operational state is observed and directed.

## Implementation Notes
The concept of Deck is realized as a web-based dashboard. Implementation details—including layout, panels, data model, and real-time behavior—are documented in the [[deck]] app note.
