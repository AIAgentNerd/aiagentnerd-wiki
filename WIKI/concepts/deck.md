# Concepts Deck

## Summary
Deck is the primary control interface for the AiAgentNerd multi-agent system. It provides a unified operational view using a ship metaphor, allowing users to visualize agents, monitor tasks, manage users, and observe real-time system activity.

## Key Concepts
- **Ship metaphor**: AiAgentNerd is the ship, Deck is the control interface, agents are the crew, tasks are missions, the activity feed is the ship log, and system monitoring is the ship’s status.
- **Core capabilities**: Agent visualization, task monitoring, user management, and system activity observation.
- **System role**: Deck serves as the main control surface, the visualization layer for Hermes task execution, and the operator’s entry point into the system.

## Practical Use
- See the current status and activity of all agents at a glance.
- Track active and queued tasks as they progress.
- Manage users and their permissions to the system.
- Follow a chronological event feed (ship log) to debug or audit system actions.

## Implementation Notes
- Deck is implemented as a web-based dashboard; technical details are documented in the [[apps/deck]] app note.
