# Openwebui New Chat 538a6d4c 495a 4807 93a4 549df022cb7f

## Summary
The AiAgentNerd knowledge system ingests unstructured text, automatically structures it, and evolves stored knowledge through merging and cleanup rather than duplication. This note records the operator-facing command set, safety conventions, and daily workflows for interacting with the ingestion, merge, and cleanup pipelines.

## Key Concepts
- **Automatic Ingestion**: Raw notes, transcripts, chat exports, and ideas are accepted; the system assigns categories, generates filenames, and structures content without manual path selection.
- **Structured Output Format**: Ingested material is organized into standard sections: topics, context, key concepts, steps, commands, and important details.
- **Merge-First Evolution**: New information is merged into existing knowledge where possible; the system identifies canonical files and avoids duplicates.
- **Knowledge Cleanup Agent**: Background scan that detects duplicate files, similar topics, and low-value notes, grouping them with confidence scores and recommended actions.
- **Preview-Confirm Safety Gate**: All writes, merges, archives, and deletions require an explicit preview followed by operator confirmation; backups are created before changes.
- **Archive Preferred Over Delete**: Cleanup recommends archiving; deletion is restricted to high-confidence cases and requires additional confirmation.

## Practical Use
- **Ingest with automatic save**: Paste raw content and issue `Clean this up and save it`.
- **Ingest with preview**: Paste raw content and issue `Save this to knowledge with preview`; review the output, then finalize with `confirm` or abort with `cancel`.
- **Format without saving**: Paste content and issue `Clean this up` to receive structured output without persistence.
- **Merge multiple chunks**: Paste separate chunks separated by `---`, then issue `Merge these` and confirm with `confirm merge`.
- **Topical merge**: Issue `Merge this with existing knowledge about <topic> with preview`; if multiple files match, select with `merge <n>`, then `confirm merge`.
- **Targeted file merge**: Issue `Merge this with existing file <filename>.md with preview`, then `confirm merge`.
- **Split large inputs**: Paste large content and issue `Split this into multiple notes` to decompose into discrete entries.
- **Run full cleanup**: Issue `cleanup knowledge` to scan the entire knowledge base.
- **Run scoped cleanup**: Issue `cleanup knowledge about <topic>` to limit the scan.
- **Merge cleanup duplicates**: After cleanup review, issue `merge cleanup group <n> into canonical with preview`, then `confirm cleanup merge`.
- **Archive cleanup group**: Issue `archive cleanup group <n>`, then `confirm archive`.
- **Delete cleanup group**: Issue `delete cleanup group <n>` (restricted to high-confidence matches only).
- **Cancel any operation**: Use `cancel`, `cancel merge`, or `cancel cleanup` to abort the active workflow.

## Implementation Notes
- The ingestion pipeline automatically categorizes content and writes to the appropriate repository path; operators should avoid manual path selection during normal use.
- Confirmation commands (`confirm`, `confirm merge`, `confirm cleanup merge`, `confirm archive`) are required state transitions before any mutating operation is executed.
- The cleanup agent does not modify files autonomously; it surfaces grouped recommendations and awaits explicit operator action.
- Backup snapshots are captured automatically before merge, archive, and delete operations.
- Daily workflow follows a three-phase pattern: capture (ingest), improve (merge into existing), and combine (merge related ideas).
