# Openwebui New Chat 538a6d4c 495a 4807 93a4 549df022cb7f

## Summary
Documents the AiAgentNerd knowledge system ingestion, merge, and cleanup workflows. Covers how unstructured input is transformed into structured knowledge, the preview-confirm-save safety model, and the Knowledge Cleanup Agent's duplicate detection and archival processes.

## Key Concepts
- **Automatic Ingestion**: Accepts raw notes, transcripts, chat outputs, and unstructured thoughts; transforms them into structured knowledge entries
- **Automatic Organization**: System assigns categories, generates filenames, structures content, and routes to the correct storage location without manual intervention
- **Structured Output Format**: Every entry is organized into topics, context, key concepts, steps (if applicable), commands (if applicable), and important details
- **Knowledge Merging**: New information is merged into existing knowledge rather than creating duplicates; supports topic-based and file-specific merging
- **Preview-Confirm-Save Safety Model**: All destructive or modifying operations require explicit operator confirmation after a preview; backups are created before changes
- **Knowledge Cleanup Agent**: Scans the knowledge base to identify duplicates, similar topics, and low-value or noisy notes; presents grouped recommendations for review
- **Cleanup Actions**: Merge into canonical, archive, or ignore; automatic deletion is disabled and archive is preferred over delete

## Practical Use
- **Save new knowledge**: `Clean this up and save it` → paste content. For preview mode: `Save this to knowledge with preview` → review → `confirm`
- **Clean without saving**: `Clean this up` → paste content
- **Merge multiple raw pieces**: `Merge these` → paste chunks separated by `---` → `confirm merge`
- **Improve existing knowledge by topic**: `Merge this with existing knowledge about <topic> with preview` → paste content → if multiple files match, select with `merge <n>` → `confirm merge`
- **Merge into specific file**: `Merge this with existing file <filename>.md with preview` → paste content → `confirm merge`
- **Split oversized content**: `Split this into multiple notes` → paste large content
- **Cancel operations**: `cancel`, `cancel merge`

## Implementation Notes
- **Cleanup invocation**: `cleanup knowledge` or `cleanup knowledge about <topic>`
- **Cleanup review output**: Returns grouped items (duplicate group, low-value group, similar topic group) with files, confidence scores, and recommendations
- **Cleanup merge**: `merge cleanup group <n> into canonical with preview` → `confirm cleanup merge`
- **Cleanup archive**: `archive cleanup group <n>` → `confirm archive`
- **Cleanup delete**: `delete cleanup group <n>` — restricted to high-confidence cases only; requires confirmation
- **Safety constraints**: No automatic deletion; archive preferred over delete; confirmation required for all mutations; backups created before changes
- **Operator best practices**: Prefer merge over duplicate creation; use preview for important content; keep input raw and let the system handle cleanup
