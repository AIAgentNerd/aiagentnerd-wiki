# Openwebui New Chat 538a6d4c 495a 4807 93a4 549df022cb7f

## Summary
Reference document for the AiAgentNerd knowledge system's ingestion, organization, merge, and cleanup behaviors. Defines the operator command patterns and safety constraints for daily knowledge management workflows.

## Key Concepts
- **Ingestion pipeline**: Accepts unstructured input (notes, transcripts, chat logs, ideas) and automatically extracts structured fields: topics, context, key concepts, steps, commands, and important details.
- **Automatic organization**: System assigns categories, generates filenames, structures content, and selects storage locations without manual filing.
- **Knowledge merging**: New information is integrated into existing canonical files rather than creating duplicates. Existing knowledge is updated and refined continuously.
- **Knowledge Cleanup Agent**: Background process that scans the knowledge base to detect duplicate files, similar topics, and low-value notes. Outputs grouped recommendations with confidence scores.
- **Safety model**: All mutations require explicit confirmation after preview. Archives are preferred over deletions. Backups are created before changes. No automatic deletions occur.
- **Evolutionary improvement**: Accumulated input improves structural quality and output relevance over time.

## Practical Use
- **Save new knowledge**: Operator command `Clean this up and save it` followed by raw content. For preview mode: `Save this to knowledge with preview`, then `confirm` or `cancel`.
- **Clean only (no persistence)**: `Clean this up` followed by raw content.
- **Merge into topic**: `Merge this with existing knowledge about <topic> with preview`. If multiple candidates exist, select with `merge <number>`, then `confirm merge`.
- **Merge into specific file**: `Merge this with existing file <filename>.md with preview`.
- **Split large input**: `Split this into multiple notes` followed by the content block.
- **Initiate cleanup**: `cleanup knowledge` (full scan) or `cleanup knowledge about <topic>` (scoped scan).
- **Cleanup actions**:
  - Merge duplicates: `merge cleanup group <n> into canonical with preview` → `confirm cleanup merge`
  - Archive: `archive cleanup group <n>` → `confirm archive`
  - Delete (restricted): `delete cleanup group <n>` (high-confidence cases only)
  - Cancel: `cancel cleanup`
- **General control verbs**: `confirm`, `cancel`, `confirm merge`, `cancel merge`.

## Implementation Notes
- Structured output schema for ingested content includes: topics, context, key concepts, steps (conditional), commands (conditional), important details.
- Cleanup output groups: duplicate group, low-value group, similar topic group. Each entry includes file references, confidence level, and recommended action (merge, archive, ignore).
- All cleanup recommendations are surfaced for operator review; zero-touch automation is not enabled for destructive actions.
- Deletion gating requires high-confidence detection to prevent accidental data loss.
- Operator pattern: feed raw, unformatted input to the system rather than manually authoring structured notes.
