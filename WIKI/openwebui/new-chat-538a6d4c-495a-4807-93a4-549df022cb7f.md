# Openwebui New Chat 538a6d4c 495a 4807 93a4 549df022cb7f

## Summary
Documentation of the AiAgentNerd knowledge ingestion, merge, and cleanup workflows. Covers the user-facing command vocabulary, confirmation gates, and safety constraints that govern how raw input is transformed into structured canonical knowledge without manual file management.

## Key Concepts
- **Ingestion Pipeline**: Raw input (notes, transcripts, chat outputs, thoughts) is automatically cleaned, categorized, assigned a filename, and stored in the correct location
- **Merge-First Policy**: Prefer updating existing canonical knowledge over creating duplicates; the system suggests relevant existing files during ingestion
- **Cleanup Agent**: Scans the knowledge base to identify duplicates, similar topics, and low-value notes, grouping them into review queues with confidence scores and recommendations
- **Confirmation Gates**: All mutating operations require explicit approval via `confirm`, `confirm merge`, `confirm cleanup merge`, or `confirm archive`; previews are available before persistence
- **Archive-First Policy**: Destructive changes are avoided; cleanup prefers merge or archive over deletion, and backups are created before any change
- **Topic-Centric Model**: Users think in topics rather than files; the system manages underlying structure automatically

## Practical Use
- **Capture raw input**: `Clean this up and save it` (auto-save) or `Save this to knowledge with preview` (requires `confirm`)
- **Clean without saving**: `Clean this up`
- **Merge multiple raw chunks**: `Merge these` followed by `confirm merge`
- **Update existing topic**: `Merge this with existing knowledge about <topic> with preview`, then `confirm merge` (if multiple files appear, use `merge 1`, etc.)
- **Update specific file**: `Merge this with existing file <filename>.md with preview`
- **Split large content**: `Split this into multiple notes`
- **Cancel operation**: `cancel`, `cancel merge`, or `cancel cleanup`
- **Run cleanup**: `cleanup knowledge` or `cleanup knowledge about <topic>`
- **Review cleanup groups**: Groups are returned as duplicate, low-value, or similar topic clusters, each listing files, confidence, and recommendation
- **Resolve cleanup group**: `merge cleanup group <n> into canonical with preview` then `confirm cleanup merge`; or `archive cleanup group <n>` then `confirm archive`
- **Delete (restricted)**: `delete cleanup group <n>` only for high-confidence cases
- **Daily flow**: capture → improve (merge with existing) → combine (merge related ideas)

## Implementation Notes
- The system does not perform automatic deletions or overwrites; every mutating action stops at a preview/confirmation gate
- Cleanup operates in read-only scan mode until the user selects an action; it suggests a canonical file for each duplicate group
- Auto-assigned metadata includes category, filename, and structure (topics, context, key concepts, steps, commands, details)
- Deletion commands are gated and only presented when confidence is high; the default resolution path is merge or archive
- Backups are created before applying merge, archive, or delete actions
