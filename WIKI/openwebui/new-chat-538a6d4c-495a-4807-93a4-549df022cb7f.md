# Openwebui New Chat 538a6d4c 495a 4807 93a4 549df022cb7f

## Summary
The AiAgentNerd knowledge system is an automated ingestion and maintenance pipeline that converts raw, unstructured input into structured technical wiki notes. It handles categorization, filename generation, schema enforcement, iterative merging with existing knowledge, and periodic deduplication via a cleanup agent. All mutating operations are gated by preview and explicit operator confirmation.

## Key Concepts
- **Ingestion Pipeline**: Accepts arbitrary raw text—notes, transcripts, chat logs, ideas—and automatically cleans, categorizes, filenames, and routes it to the correct knowledge repository.
- **Auto-Structuring**: Enforces a consistent output schema including topics, context, key concepts, steps, commands, and important details to ensure reusability.
- **Merge Engine**: Preferentially updates existing notes rather than creating duplicates. Supports topic-based, file-specific, and multi-source merging.
- **Knowledge Cleanup Agent**: Scans the knowledge base to identify duplicates, near-duplicate topics, and low-value notes; groups them by confidence and recommends merge, archive, or ignore actions.
- **Safety Gate**: Destructive or mutating operations require a preview stage and explicit confirmation. Backups are created before changes, and archival is preferred over deletion.
- **Control Interface**: Operated via explicit natural-language command phrases interpreted by the system orchestrator.

## Practical Use
### Ingestion Commands
- `Clean this up and save it` — Ingest raw input directly without preview.
- `Save this to knowledge with preview` — Ingest with preview; requires `confirm` to persist.
- `Clean this up` — Clean and structure only; do not save.
- `Split this into multiple notes` — Decompose large input into separate notes.

### Merge Commands
- `Merge these` — Combine multiple pasted chunks; requires `confirm merge`.
- `Merge this with existing knowledge about <topic> with preview` — Topic-based update; review then `confirm merge`.
- `Merge this with existing file <filename>.md with preview` — Targeted file update; review then `confirm merge`.
- `confirm merge` / `cancel merge` — Finalize or abort a pending merge.

### Cleanup Workflow
1. Initiate: `cleanup knowledge` or `cleanup knowledge about <topic>`.
2. Review output groups: duplicate group, low-value group, similar-topic group. Each lists files, confidence, and recommendation.
3. Resolve:
   - `merge cleanup group <n> into canonical with preview` → `confirm cleanup merge`
   - `archive cleanup group <n>` → `confirm archive`
   - `delete cleanup group <n>` — Restricted to high-confidence cases only.
   - `cancel cleanup` — Abort the cleanup session.

### General Control
- `confirm` / `cancel` — Standard confirmation gates for previewed operations.

### Daily Operator Flow
1. **Capture**: `Clean this up and save it`
2. **Improve**: `Merge this with existing knowledge about <topic> with preview`
3. **Combine**: `Merge these` (related ideas)

## Implementation Notes
- **Automatic Routing**: The system selects the target repository (`aiagentnerd-wiki`, `aiagentnerd-system`, or `goldienerd-knowledge`), category folder, and filename without manual intervention.
- **Schema Enforcement**: Ingested output is always structured with frontmatter and consistent sections to maintain wiki compatibility.
- **Deduplication Behavior**: The cleanup agent performs similarity detection across the knowledge base and suggests a canonical file for each group. No automatic writes or deletions occur.
- **Safety Controls**:
  - Preview mode is available for all save and merge operations.
  - Backups are created before any mutating change.
  - Archive is the default removal path; hard deletion is gated and only offered for high-confidence low-value items.
- **Evolution Model**: Repeated ingestion of related topics is expected to refine and expand existing canonical notes rather than fragment storage.
