# Openwebui New Chat 538a6d4c 495a 4807 93a4 549df022cb7f

## Summary
The AiAgentNerd knowledge system provides an automated ingestion, structuring, and evolution pipeline for raw information. It converts unstructured input into organized, reusable knowledge through automatic categorization, filename generation, and structured formatting, while a cleanup agent maintains quality over time via merge and archive workflows with mandatory confirmation.

## Key Concepts
- **Automatic ingestion**: Raw input (notes, transcripts, chat outputs, ideas) is cleaned and structured without manual filing decisions
- **Structured output format**: Every input is organized into topics, context, key concepts, steps, commands, and important details
- **Merge workflow**: New information is merged into existing knowledge rather than creating duplicates, enabling continuous evolution
- **Knowledge Cleanup Agent**: Scans the knowledge base to detect duplicates, similar topics, and low-value notes, grouping them for review
- **Safety-first design**: Preview → confirm → save pattern prevents accidental overwrites; archive preferred over deletion; backups created before changes
- **Topic-oriented organization**: Knowledge is organized by topic rather than by individual files

## Practical Use
- **Ingestion commands**:
  - `Clean this up and save it` — standard ingestion
  - `Save this to knowledge with preview` — ingestion with preview step
  - `Clean this up` — clean only, no save
- **Merge commands**:
  - `Merge these` — merge multiple pasted chunks
  - `Merge this with existing knowledge about <topic> with preview` — update existing topic
  - `Merge this with existing file <filename>.md with preview` — target specific file
  - `confirm merge` / `cancel merge` — finalize or abort
- **Splitting**:
  - `Split this into multiple notes` — break large content into separate notes
- **Cleanup commands**:
  - `cleanup knowledge` — full knowledge base scan
  - `cleanup knowledge about <topic>` — scoped scan
  - `merge cleanup group <n> into canonical with preview` — deduplication
  - `archive cleanup group <n>` — archive low-value group
  - `delete cleanup group <n>` — restricted deletion (high-confidence only)
  - `confirm cleanup merge` / `confirm archive` / `cancel cleanup` — finalize actions
- **Daily workflow**: Capture → Improve (merge with existing) → Combine (merge related ideas)
- **Safety constraints**: No automatic deletion; all destructive actions require explicit confirmation; backups are created before changes

## Implementation Notes
- **Command interface**: The system is operated via natural-language triggers (e.g., `Clean this up and save it`, `Merge this with existing knowledge about <topic> with preview`). These function as the primary user-facing control surface.
- **Output schema**: Ingested content is structured into: topics, context, key concepts, steps (if procedural), commands (if relevant), and important details.
- **Storage automation**: The system auto-selects category, generates filename, structures content, and places it in the correct repository (`aiagentnerd-wiki`, `aiagentnerd-system`, or `goldienerd-knowledge` based on content type).
- **Cleanup agent behavior**: Scans are read-only until operator action. The agent groups findings into duplicate groups, similar-topic groups, and low-value groups, each with confidence scores and recommended actions (merge, archive, ignore).
- **Safety mechanisms**: 
  - Preview → confirm → save pattern on all writes
  - Archive preferred over delete; no automatic deletion
  - Backups created before merge/archive operations
  - Delete command restricted to high-confidence cases only
