## Implementation Notes
- **Ingestion Process**:
  - The system automatically categorizes, generates filenames, and structures content.
  - It transforms messy input into topics, context, key concepts, steps, commands, and important details.
- **Merge Process**:
  - New information is merged into existing files, updating and improving them.
  - The system prevents accidental overwrites and unintended changes by requiring confirmation.
- **Cleanup Process**:
  - The Knowledge Cleanup Agent scans the entire knowledge base, identifies duplicates, similar topics, and low-value content.
  - It groups these for review, suggesting actions like merging, archiving, or ignoring.
  - No changes are made automatically; all actions require confirmation.
- **Safety Features**:
  - Backups are created before any changes.
  - Archive is preferred over delete, and only high-confidence deletions are allowed.
- **Daily Usage**:
  - Follow the simple workflow to capture, improve, and combine knowledge.
  - Use the provided commands to interact with the system efficiently.
- **Best Practices**:
  - Do not overthink categories; let the system decide.
  - Prefer merging over creating duplicates.
  - Use preview for important content.
  - Think in topics, not files.
  - Keep input raw and let the system handle cleanup.
