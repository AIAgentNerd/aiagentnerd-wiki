## ASSISTANT

Below is a straightforward, reusable layout you can use as a starting point for any **ACR (Automatic Content Recognition) UI** – whether that’s for a media‑tracking dashboard, a podcast metadata extractor, or an asset‑management console.  
Feel free to adapt the component names and structure to match your exact use‑case.  

| Component | Purpose | Suggested UX |
|-----------|---------|--------------|
| **Header** | Global navigation & branding | Logo left, title “ACR Dashboard”, user‑menu right (profile, logout) |
| **Sidebar** | Contextual navigation | • **Dashboard** (overview) <br>• **Jobs** (queue, history) <br>• **Settings** <br>• **Help / Docs** |
| **Main Area** | Core content | Split into two vertical panels: 1️⃣ **Control Pane** – start/stop/refresh jobs, filters, display of selected job’s status; 2️⃣ **Results Pane** – table/grid of detected content, timestamps, confidence scores, media preview. |
| **Footer** | Status & quick actions | • Sync status indicator <br>• Snack‑bars for toast messages <br>• Small “About” link |

### Design Tips

1. **Responsive Grid** – Use CSS‑grid or flexbox so the Control & Results panels stack on narrow viewports.
2. **Dark/Light Mode** – Give users a toggle; most UI libraries (MUI, Ant Design, Chakra) have built‑in support.
3. **Accessibility** – Ensure proper `aria` labels for menus, buttons, and table cells.
4. **State Feedback** – Show spinners or progress bars next to running jobs, and use toasters for success/failure messages.
5. **Data Export** – Add a button in the Results pane for CSV/PDF export, useful for audit trails.
